# Website Section Refinement

Use this reference when redesigning one part of a public website.

## Workflow

1. Identify the section role:
   - orientation
   - persuasion
   - product proof
   - feature explanation
   - comparison
   - social proof
   - lead capture
   - navigation
   - closure
2. Preserve the page's global aesthetic direction and memory point.
3. Read `templates/website-sections/index.yaml`, then the relevant section template.
4. Refine the section's:
   - information hierarchy
   - copy specificity
   - visual rhythm
   - media/product signal
   - CTA priority
   - responsive behavior
   - accessibility behavior
5. Check the section in page context so it does not feel like an unrelated ad block.

## Website Section Rules

- Website sections should create a story sequence, not a stack of generic cards.
- Every section should answer a visitor question or move the visitor closer to action.
- Use real product, offer, proof, or story signals whenever possible.
- Do not hide the product behind abstract gradients or decorative screenshots.
- Avoid dashboard/sidebar/table patterns unless the site is explicitly demonstrating a product UI.

## Section Quality Bar

A section is strong when it has:

- one clear job
- one dominant visual idea
- specific copy
- a visible relationship to the global style
- mobile behavior that preserves the message
- an action, proof, or transition into the next section

## Common Website Fixes

- If a hero is generic, make the product or offer the first-viewport signal.
- If features are interchangeable, rewrite them around user outcomes and proof.
- If testimonials feel fake, add role, context, result, or source detail.
- If pricing cards are generic, add fit cues and tradeoffs.
- If a CTA band is weak, make the next step concrete and low-friction.
- If a footer is neglected, use it for trust, navigation, and contact confidence.
